# Technical Interview

## What is a Technical Interview?
For a Computer Science position, a technical interview is typically the first step in the interviewing process. These involve solving algorithmic questions well in a limited amount of time (typically around an hour). This acts as a filter which most applicants do not pass. It is important to be particularly well versed in algorithms for these interviews.

## Format
They will usually provide a place to write and test code along with a few example problems. You will be graded not only on your code, but also how well you are able to communicate your problem solving process. 

## Preparation
For the quickest, most direct practice, I would recommend the <a href="neetcode.html">NeetCode 150</a>. These are questions that every applicant should learn how to do. Try explaining each problem to yourself as you do it. This will help you explain your thought process, which you will have to do in an actual interview. For more practice after completing the NeetCode 150, check out <a href="https://leetcode.com/" target="_blank">LeetCode</a>, which has 3000+ algorithmic questions, often times taken from actual interviews. Also brush up on <a href="https://www.w3schools.com/dsa/dsa_timecomplexity_theory.php" target="_blank">time complexity</a> and <a href="https://www.theodinproject.com/lessons/javascript-space-complexity" target="_blank">space complexity</a>, as you will be expected to know how to identify each of these for the algorithms you create.

## Approaching a Problem during an Interview
1. Restate a problem and ask any clarifying questions.
2. Walk through an example by hand.
3. Propose an approach.
4. Code, explaining your process as you go.
5. Test the provided test cases and check for any edge cases that may fail.
6. Debug until all test cases pass.
7. Discuss the time / space complexity of your algorithm.
